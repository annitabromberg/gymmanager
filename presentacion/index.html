Fundamentos de Informática

GymManager: una aplicación web para gestionar un gimnasio

Presentación realizada por: Anna Bromberg, María Josefina Velázquez Paz y Ángela Romero.

Esta presentación explica, paso a paso y con lenguaje sencillo, cómo se construyó esta aplicación web y para qué sirve.

Objetivo: explicar la idea sin asumir que el público ya sabe programar.

1. ¿De qué se trata la aplicación?

GymManager es una aplicación web pensada para ayudar a un gimnasio a organizar su información. Fue desarrollada como proyecto para la asignatura Fundamentos de Informática.

Lo que puede hacer

  • Registrar alumnos
  • Guardar rutinas
  • Controlar asistencia
  • Gestionar cuotas

Idea sencilla

Si una persona quiere administrar un gimnasio, esta app sirve como una “mesa de trabajo digital” para guardar y consultar datos.

2. ¿Qué herramientas se usaron?

  • Python: lenguaje principal para crear la lógica.
  • Flask: framework para construir páginas web.
  • HTML: define la estructura de las páginas.
  • CSS: cambia el diseño y la apariencia visual.
  • JSON: guarda la información de alumnos, rutinas y asistencia.

En otras palabras: se usó una combinación de lenguaje, diseño y almacenamiento de datos.

3. ¿Cómo se organizó el proyecto?

Carpetas importantes

  • app.py: inicia la aplicación.
  • templates/: contiene las páginas HTML.
  • static/: contiene imágenes, videos y estilos CSS.
  • gymmanager_app/: agrupa la lógica principal.

La idea

La app está separada en partes. Cada parte cumple una tarea específica.

4. Estructura del código, explicado para principiantes

Si lo vemos como una casa, cada archivo cumple una función distinta:

app.py

Es el “motor” que prende la aplicación. Aquí se inicia la app y se define el puerto.

gymmanager_app/__init__.py

Es la “base” donde se crea la aplicación Flask y se conectan las páginas y archivos estáticos.

gymmanager_app/routes.py

Aquí están las rutas de la web, por ejemplo: login, inicio, alumnos, rutinas, asistencia.

gymmanager_app/data.py

Es el “archivo de almacenamiento”: lee y guarda la información de los alumnos y demás datos en JSON.

Es como una organización por tareas: una parte arranca la app, otra define rutas, otra guarda los datos y otra muestra las páginas.

5. Cómo se conecta todo

El flujo de una petición con la app es muy simple:

  1. El usuario abre una URL en el navegador.
  2. Flask identifica la ruta, por ejemplo /alumnos.
  3. La ruta llama una función del código.
  4. La función revisa datos, procesa información y luego devuelve una página HTML.
usuario abre /inicio -> Flask responde con la página correcta

7. Paso a paso: cómo funciona la aplicación

  1. El usuario entra a la página.
  2. Se inicia el servidor con Flask.
  3. La aplicación muestra una pantalla de login.
  4. Si el usuario escribe bien sus credenciales, entra al sistema.
  5. Desde ahí puede ver el inicio, alumnos, rutinas y asistencia.
Pensalo como una recepción de un gimnasio: primero pasa por la puerta de entrada, luego accede al control principal.

5. El login: la primera puerta

Cuando una persona se conecta, el sistema valida si el usuario y la contraseña son correctos.

usuario = "admin"
password = "1234"

Si los datos coinciden, se “abre” la sesión y el usuario puede navegar dentro de la aplicación.

6. ¿Qué pasa con los datos?

La información no se guarda en una base de datos grande como MySQL, sino en archivos JSON.

  • alumnos.json: lista de alumnos
  • rutinas.json: rutinas de entrenamiento
  • asistencia.json: control de presencia

JSON es una forma sencilla de guardar información en formato parecido a una tabla o lista.

8. ¿Qué hace cada parte del proyecto?

Flask / rutas

Define qué ocurre cuando se visita una URL como /inicio o /alumnos.

Templates

Son las páginas HTML que muestran el contenido al usuario.

CSS

Hace que la aplicación se vea bonita y moderna.

JSON

Guarda los registros que la app necesita.

9. Cómo se ve el flujo de la app

Un ejemplo de uso real:

  1. Se agrega un alumno con nombre, edad, objetivo y plan.
  2. Ese alumno queda guardado en el archivo JSON.
  3. Luego, desde la interfaz, se puede buscar, editar o eliminar.
  4. La aplicación vuelve a mostrar la información actualizada.
Esto se llama flujo de datos: la información entra, se procesa y vuelve a mostrarse al usuario.

10. ¿Por qué esta app es útil?

  • Ayuda a organizar información de forma rápida
  • Reduce errores manuales
  • Facilita la administración de un gimnasio
  • Sirve como base para aprender proyectos web reales

11. Lo más importante para aprender

La idea principal no es solo “hacer una app”, sino entender estos conceptos:

  • Una página web se compone de front-end y back-end
  • El navegador pide información y el servidor responde
  • Los datos pueden guardarse en archivos o en bases de datos
  • El diseño ayuda a que una app sea fácil de usar

12. Conclusión

GymManager es una aplicación web simple, pero con una gran idea: organizar la gestión de un gimnasio usando tecnología accesible.

Para principiantes, esta app sirve como ejemplo perfecto de cómo una idea real puede convertirse en una solución digital paso a paso.
